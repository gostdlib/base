# Tuple

**EXPERIMENTAL**

This provides a generator for variable length generic tuples.  This is considered an experimental package and I may remove it.  Go has survived without tuples forever and I only find one limited use case that I'm interested in.

That use case is map lookups by key. Sometimes I find myself doing:

map[string]map[string]value

When I could do:

```go
type lastFirst struct{
  last, first string
}

lookup := map[lastFirst}value
```

This let's me quickly create a type for doing that with X lookup parameters.  

//go:generate tuple lastFirst string, string
or 
//go:generate tuple lastFirst last:string, first:string

Where the second form creates a named tuple with Last() instead of V0 and First() instead of v1. While I used `//go:generate` , it is probably better form to just generate by running the command instead if the values are going to remain fixed sized.

I eliminated any idea of All() or Chan() with this, because if I want to just access a read only array, I can do:

```go
immutable.NewSlice([]any{"what", "ever", 1, "want"})
```

I could achieve a named tuple like so:

```go
immutable.NewMap(map[string]any{"key0": value0, "key1": value1})
````

But the advantage here is you get a `struct` that is not heap allocated and access does not use the `any` type. But if you are going to do iteration of values, then immutable versions of `slice` and `map` are better.

## Install

The tool is registered in this module's `go.mod` `tool` block. In another module,
add it as a tool dependency first:

```bash
go get -tool github.com/gostdlib/base/values/generators/tuple
```

## Usage

The name is used **verbatim**, so its case controls whether the generated type is
exported: `lastFirst` generates an unexported `lastFirst`, while `LastFirst`
generates an exported `LastFirst`. The accessor methods are exported either way.

Each field after the name is a comma-separated `type` or `name:type`
specification. A bare `type` produces a positional field with accessor `V0`,
`V1`, …; a `name:type` field replaces the positional accessor with an exported
accessor derived from the name. The two forms may be mixed.

### Flags

| Flag | Description |
|------|-------------|
| `-p` | Write a complete Go file (package clause and imports) into the current package as `<name>.go` (lower-cased). Without it, only the type and methods are printed to stdout. |

## Generated code

`tuple -p lastFirst last:string, first:string` in package `person` produces
`lastfirst.go`:

```go
// Code generated by "tuple -p lastFirst last:string, first:string"; DO NOT EDIT.

package person

import "fmt"

// lastFirst is a tuple holding 2 values: last string, first string.
type lastFirst struct {
	last  string
	first string
}

// NewlastFirst creates a new lastFirst tuple with the given values.
func NewlastFirst(last string, first string) lastFirst {
	return lastFirst{last: last, first: first}
}

// Last returns the first value of the tuple.
func (t lastFirst) Last() string {
	return t.last
}

// First returns the second value of the tuple.
func (t lastFirst) First() string {
	return t.first
}

// Len returns the number of values in the tuple.
func (t lastFirst) Len() int {
	return 2
}

// String implements fmt.Stringer.
func (t lastFirst) String() string {
	return fmt.Sprintf("(%v, %v)", t.last, t.first)
}
```

Without `-p`, the same invocation prints just the type and its methods (no
package clause or imports) to stdout, for pasting into an existing file.
